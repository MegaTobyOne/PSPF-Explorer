# PSPF 2025 Technology Domain Requirements

| ID | Title | Requirement Description |
| :-- | :-- | :-- |
| 84 | ISM Cyber Principles Throughout Lifecycle | Technology Asset Owners ensure ASD ISM cyber security principles are applied throughout the technology asset lifecycle. [^1] |
| 85 | ISM Controls and Guidelines by Risk | ASD ISM controls and guidelines are applied on a risk-basis to technology assets, considering data classification, system criticality, and threat environment. [^1] |
| 211 | Asset Stocktake \& Security Risk Plan | Maintain a Technology Asset Stocktake and a Technology Security Risk Management Plan for all internet-facing or significant technology assets. [^1] |
| 86 | System Risk Authorization | Each technology system is authorised to operate based on completion of a risk acceptance process and approval by an appropriate risk owner. [^1] |
| 87 | Authorisation via ISM Risk-Based Assessment | Authorisation decisions must be based on outcomes of ISM-advised, risk-based security assessments. [^1] |
| 88 | Authorisation for Highest Classification | Authorisations are granted to the system’s highest data classification or sensitivity. [^1] |
| 89 | Register of Authorised Systems | Maintain a register of all authorised technology systems and their approval status. [^1] |
| 90 | Reassessment after Significant Change | Systems must be reassessed for suitability if there is a significant change to the operation, threat landscape, or data holdings. [^1] |
| 91 | TikTok Ban on Government Devices | Block the TikTok application on government technology devices; exceptions require CISO/CSO approval and risk mitigation. [^1] |
| 92 | CISO/CSO TikTok Exception Handling | CISO/CSO must approve and document mitigation controls for any permitted TikTok exceptions. [^1] |
| 93 | Temporary Mitigations on Legacy IT | Apply ASD-approved temporary mitigation strategies to legacy or unsupported IT assets until replacement. [^1] |
| 94 | Secure Storage of Tech Assets | Store technology assets in facilities with appropriate security zones matching the asset's highest classification or sensitivity. [^1] |
| 95 | Secure Racks for TS Assets | Store TOP SECRET IT assets in SCEC/ASIO-prescribed secure racks or compartments when not in use. [^1] |
| 96 | ISM Disposal/Wipe Controls | Dispose of or wipe government technology assets according to ISM requirements on decommission, destruction, or change of use. [^1] |
| 97 | Mandatory Patch Management | Apply vendor-issued/ASD-recommended updates/patches within specified timeframes. [^1] |
| 98 | Minimum Essential Eight Level | Achieve and maintain Essential Eight Maturity Level 3 on externally-facing/significant systems unless risk accepted by CISO/CSO. [^1] |
| 99 | Daily Backup and Restore Testing | Perform daily backups and restoration testing for technology assets managing important/critical information. [^1] |
| 100 | DNS Filtering on Government Systems | Implement DNS filtering controls for government technology devices accessing the internet. [^1] |
| 101 | Cloud Provider Certification Requirement | Use only ASD-certified cloud/data centre providers for classified/government-significant systems, unless risk accepted by CISO/CSO. [^1] |
| 102 | Cloud Service Provider IRAP Assessment | Obtain ASD IRAP security assessment report for any Cloud Service Provider storing, processing, or transmitting government data. [^1] |
| 103 | Vulnerability Disclosure Program | Entities must participate in the ASD-coordinated government-wide Vulnerability Disclosure Program. [^1] |
| 104 | ASD Cyber Partnership Participation | Mandatory government entity registration/participation in ASD Cyber Partnership Program for vulnerability/threat sharing. [^1] |
| 214 | Gateways and SSE for Digital Infrastructure | Protect digital infrastructure by implementing either ASD-approved Gateways or Security Service Edge solutions. [^1] |
| 212 | Post-Quantum Cryptography | Use only ASD-approved post-quantum cryptographic equipment/software for handling Top Secret data or as directed by ASD. [^1] |
| 213 | Special Protection for Gov-Significant Systems | Apply additional ASD-defined cyber protections to technology assets essential to government continuity or national interest. [^1] |

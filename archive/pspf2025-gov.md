# PSPF 2025 Governance Domain Requirements

| ID | Title | Requirement Description |
| :-- | :-- | :-- |
| 1 | DoS Security Support | The Department of State supports portfolio entities to achieve and maintain an acceptable level of protective security through advice and guidance on government security. [^1] |
| 2 | Accountable Authority Direction Compliance | The Accountable Authority complies with all Protective Security Directions. [^1] |
| 3 | Technical Authority Entity Security Advice | The Technical Authority Entity provides technical advice and guidance to support entities to achieve and maintain an acceptable level of protective security. [^1] |
| 4 | Security Service Provider Support | The Shared Service Provider Entity provides security services to support entities to achieve and maintain acceptable protective security. [^1] |
| 5 | Documented Service Partnership Responsibilities | The Shared Service Provider Entity documents responsibilities and accountabilities for partnerships and service arrangements with other Australian Government entities. [^1] |
| 6 | Minister Accountability | The Accountable Authority is answerable to the entity’s Minister for the entity’s protective security arrangements. [^1] |
| 7 | Authority Manages Entity Security Risk | The Accountable Authority manages the security risks of the entity. [^1] |
| 8 | CSO Appointment and Empowerment | The Accountable Authority appoints and empowers a Chief Security Officer (CSO) to lead the entity’s protective security program. [^1] |
| 9 | CSO SES/Minimum Clearance | The CSO is a Senior Executive Service Officer with a minimum NV1 (Negative Vetting 1) security clearance. [^1] |
| 10 | CSO Accountability to Authority | The CSO is accountable to the Accountable Authority for security matters in the entity. [^1] |
| 11 | CISO Appointment for Cyber Program | The Accountable Authority appoints a Chief Information Security Officer (CISO) to oversee the entity’s cyber security program and the cyber security for the entity’s most critical technology resources, including information technology and operational technology. [^1] |
| 12 | CISO Capability and Clearance | The CISO has the right capability, experience, and minimum NV1 (Negative Vetting 1) clearance for the responsibility. [^1] |
| 13 | CISO Accountability for Cyber Risk \& Program | The CISO is accountable to the Accountable Authority for cyber security risks and how the entity’s cyber security program is managing these risks. [^1] |
| 14 | Security Practitioner Resourcing | Security practitioners (where appointed) are skilled, empowered, and resourced to perform required functions. [^1] |
| 15 | Practitioner Security Training | Security practitioners (where appointed) have access to relevant training to keep abreast of emerging and evolving security issues. [^1] |
| 16 | Governance Approval Matches Entity Risk | The Accountable Authority approves security governance arrangements suitable for the entity’s size, complexity and risk. [^1] |
| 17 | Dedicated Security Email Channel | The entity establishes and monitors a dedicated security email address for security communications and notifications. [^1] |
| 18 | Security Plan Development, Implementation, Maintenance | The entity develops, implements, and maintains a security plan that addresses mandatory elements specified by this Framework. [^1] |
| 19 | Security Plan Authority Approval | The Accountable Authority approves the entity’s security plan. [^1] |
| 20 | Security Plan Annual Consideration/Review | The entity considers its security plan annually and reviews it at least every two years. [^1] |
| 21 | Security Actions/Procedures | The entity implements procedures to ensure all elements detailed in the plan are achieved. [^1] |
| 22 | Security Monitoring | The entity monitors compliance with the plan and regularly assesses the effectiveness of its security measures. [^1] |
| 23 | Security Risk Culture | The entity fosters and maintains a positive security risk culture, modelled and promoted by its leaders. [^1] |
| 24 | Security Awareness Activities | The security awareness activities meet mandatory requirements and needs of entity and staff. [^1] |
| 25 | Security Incidents: Procedures and Response | The entity develops and maintains procedures to detect, report, record, and respond to security incidents. [^1] |
| 26 | Security Incident Reporting | Significant or externally reportable security incidents are reported to the appropriate authorities per relevant government directions. [^1] |
| 27 | Security Review and Lessons Learned | After a significant incident, the entity reviews actions and outcomes, integrating lessons learned into ongoing programs. [^1] |
| 28 | Assurance Activities Follow-up | The entity undertakes and responds to assurance activities (e.g., audits), rectifying deficiencies and informing the CSO. [^1] |
| 29 | Annual Security Report Preparation | The entity prepares an annual report on performance against the PSPF and submits it to the responsible Minister and Home Affairs. [^1] |
| 30 | Cyber Health Check Survey | The entity completes the required annual ASD Cyber Security Survey for the Minister for Home Affairs. [^1] |
| 31 | Security Direction - Planning \& Implementation | Security directions are planned and implemented, with any changes recorded and actioned in the security plan. [^1] |
| 32 | Security Plan Communication | The entity communicates security plan arrangements to all relevant personnel. [^1] |
| 33 | Security Roles and Responsibilities Communication | The entity communicates the security roles, responsibilities, and accountabilities across the entity. [^1] |
| 34 | Audit Committee Security Risk Coverage | Security matters are a standing agenda item for the Audit Committee; the CISO/CSO has direct reporting capability to the Committee. [^1] |
| 35 | Protecting Whistleblowers/Critical Security Reports | Security framework protects and encourages whistleblowers and early reporters of critical security issues. [^1] |
